It was a pleasant surprise to find an entire "conventional" language underneath with loops, branching and math operations common to all imperative languages. Kernighan was a primary author, so it is no surprise it has a very C like syntax. Unlike C it is completely untyped. 有一天，我坐下来学习 Awk，这是一种以简洁而强大的单行代码着称的语言。发现一个完整的“常规”语言，其中包含所有命令式语言共有的循环、分支和数学运算，真是令人惊喜。 Kernighan 是主要作者，因此它具有非常类似于 C 的语法也就不足为奇了。与 C 不同，它是完全无类型的。 There are positional variables ($1) similar to what is found in shells. Regex can be used anywhere. Everything fits together perfectly for maximum convenience while processing text. 位置变量 ( $1 ) 类似于 shell 中的位置变量。正则表达式可以在任何地方使用。一切都完美地结合在一起，以在处理文本时提供最大的便利。 Kernighan didn't write C  Kernighan 没有写 C Yeah, be he co-wrote the book on it. 是的，他与人合着了这本书。   But this is sound, not text. So what is going on here? Sound is just a stream of numbers. Text is also a stream of numbers. If you've ever tried making white noise with cat /dev/urandom > /dev/dsp this is just an extension of the idea. In fact, the very first version of this script was 但这是声音，不是文字。那么这里发生了什么？声音只是一连串的数字。文本也是数字流。如果您曾经尝试过使用 cat /dev/urandom > /dev/dsp 制作白噪声，这只是这个想法的延伸。事实上，这个脚本的第一个版本是 cat /dev/urandom | fold -b1 | awk '{for (i=0;i<100;i++) printf($1);}' > /dev/dsp which has the effect of holding each random value for 100 cycles, lowering the pitch of the noise. While Awk was made for processing text, it has a general purpose language backing it up. To illustrate a few of these features, here is the oldest version of the script: 它具有将每个随机值保持 100 个周期的效果，从而降低噪声的音调。虽然 Awk 是为处理文本而设计的，但它有一种通用语言作为后盾。为了说明其中的一些功能，这里是最旧版本的脚本：   awk 'BEGIN {srand(); while(1) { wl=400*(0.87055^(int(rand()*10)+1)); d=(rand()*80000+8000)/wl; for (i=0;i<d;i++) { for (j=0;j<wl;j++) {printf("a")}; for (j=0;j<wl;j++) {printf("z")}; };};};' > /dev/dsp If you're using PulseAudio, you can replace the "> /dev/dsp" with... 如果您使用的是 PulseAudio，则可以将“> /dev/dsp”替换为...   This makes sound, but it pretty crude. It plays random square waves for random durations. The BEGIN{} stanza gives a chunk of code for Awk to execute before processing standard input. Sadly I don't have the previous iteration of the script, which used yes | awk '{...}' instead of BEGIN. The third line, wl=400*(0.87055^(int(rand()*10)+1)), takes a bit of explaining. A musical scale is exponential in nature, and this function generates the wavelengths of notes on a musical scale. There is one magic number, 0.87055. This is not a random number or even something found by trial and error. 这听起来不错，但它很粗糙。它在随机持续时间内播放随机方波。 BEGIN{} 节为 Awk 在处理标准输入之前执行一段代码。遗憾的是我没有脚本的前一次迭代，它使用 yes | awk '{...}' 而不是 BEGIN 。第三行 wl=400*(0.87055^(int(rand()*10)+1)) 需要一些解释。音阶本质上是指数的，此函数生成音阶上音符的波长。有一个神奇的数字，0.87055。这不是一个随机数，甚至不是通过反复试验发现的东西。 It is the fifth root of 0.5. One half is important because every octave the wavelength doubles or halves. Using the fifth root gives five intervals for each octave, a pentatonic scale. Why five notes? It is really easy to make something that sounds good in the pentatonic scale and really hard for a pair of notes to be dissonant. With a wavelength and a duration for the note, a pair of for loops pumps out a square wave. A and Z make up the low and high values. 它是 0.5 的五次方根。二分之一很重要，因为每个八度音程的波长都会加倍或减半。使用五次根为每个八度音程提供五个音程，即五声音阶。为什么是五个音符？制作五声音阶听起来不错的东西真的很容易，但要让一对音符变得不和谐真的很难。有了音符的波长和持续时间，一对 for 循环就会输出一个方波。 A 和 Z 构成低值和高值。 The more complicated version grew out of this. The added features are triangle waves, two note chords, amplitude decay, and anti-pop. The last item needs a little explaining. Towards the end of the script there is a short loop, while(c!=128), which returns the interrupted triangle wave to center. Without it there will be a sharp popping sound between note transitions. An explanation of the scoping rules regarding the rate variable is left as an exercise for the reader. 更复杂的版本由此产生。添加的功能是三角波、两个音符和弦、振幅衰减和抗流行。最后一项需要稍微解释一下。在脚本的末尾有一个短循环 while(c!=128) ，它将中断的三角波返回到中心。没有它，音符转换之间会有尖锐的爆裂声。关于 rate 变量的作用域规则的解释留作读者练习。   Since the output is a single byte, this is real 8 bit music.
